# 🚀 دليل الإعداد السريع - ثيم سُفرة

## ⚠️ مهم: لماذا يظهر الثيم الأساسي؟

**هذا طبيعي!**

ثيم سُفرة مبني على مفهوم **المكونات القابلة للإضافة** - المكونات الجديدة التي أنشأناها (Hero و Products Grid) **لا تظهر تلقائياً**.

**يجب على التاجر إضافتها من Dashboard** حسب احتياجه.

---

## 📋 خطوات تفعيل مكونات سُفرة

### الخطوة 1: تأكد من نشر الثيم

```bash
# في terminal، من داخل مجلد المشروع
cd c:\Users\diaal\Downloads\suf

# بناء الثيم
npm run production

# ربط مع المتجر (إذا لم تفعل بعد)
salla theme link

# معاينة الثيم
salla theme preview
```

### الخطوة 2: إضافة مكون Hero Section

1. **افتح Dashboard** للمتجر التجريبي
2. اذهب إلى **"الصفحة الرئيسية"** → **"إدارة الصفحة الرئيسية"**
3. اضغط زر **"+ إضافة قسم"**
4. ابحث عن **"القسم الرئيسي (Hero) - سُفرة"** 🍽️
5. اضغط عليه لإضافته
6. ستظهر لك 17 إعداد قابل للتخصيص:
   - 📸 صورة الخلفية (Desktop + Mobile)
   - ✍️ العنوان والنص الفرعي
   - 🔘 زر Call-to-Action
   - 🎨 الألوان والمحاذاة
7. خصص حسب رغبتك
8. اضغط **"حفظ"**

### الخطوة 3: إضافة مكون Products Grid

1. في نفس الصفحة، اضغط **"+ إضافة قسم"** مرة أخرى
2. ابحث عن **"قسم المنتجات (Grid/List/Menu) - سُفرة"** 🍕
3. اضغط عليه
4. ستظهر لك 13 إعداد:
   - 📦 اختيار المنتجات
   - 📐 نمط العرض (Grid / List / Menu)
   - 🖼️ نسبة الصورة
   - 👁️ إظهار/إخفاء الوصف والوسوم
5. خصص الإعدادات
6. اضغط **"حفظ"**

### الخطوة 4: تخصيص صفحة المنتج (اختياري)

1. اذهب إلى **"الإعدادات"** → **"خيارات صفحة المنتج"**
2. ابحث عن قسم **"🍽️ إعدادات المطاعم - سُفرة"**
3. ستجد 7 خيارات:
   - معرض الصور (أسفل / جانب)
   - حجم الصورة
   - بروز السعر
   - زر ثابت للموبايل
   - إظهار المكونات
   - إظهار تحذيرات الحساسية
   - إظهار المعلومات الغذائية
4. فعّل ما تريد
5. احفظ

## 🎯 النتيجة المتوقعة

بعد إضافة المكونات من Dashboard، سيظهر:

### الصفحة الرئيسية:
```
┌─────────────────────────────────────┐
│                                     │
│         Hero Section 🖼️             │
│    (صورة كبيرة + عنوان + زر)        │
│                                     │
├─────────────────────────────────────┤
│                                     │
│      Products Grid 📦               │
│   (Grid أو List أو Menu)            │
│                                     │
│  [منتج] [منتج] [منتج] [منتج]       │
│                                     │
└─────────────────────────────────────┘
```

### صفحة المنتج:
- معرض صور محسّن
- سعر بارز (إذا فعّلته)
- زر ثابت على الموبايل (إذا فعّلته)
- أقسام اختيارية (مكونات، حساسية، تغذية)

## 🐛 حل الأخطاء في Console

الأخطاء التي تظهر في Console طبيعية في مرحلة التطوير:

### 1. خطأ `salla.webp`:
هذا ملف من الثيم الأساسي - تجاهله.

### 2. أخطاء `failed to load resource`:
هذه تحدث لأن بعض الملفات من CDN الخاص بسلة.

### 3. أخطاء التوجيه (routing):
طبيعية في Preview Mode.

**✅ إذا كان الثيم يعمل ويمكنك إضافة المكونات من Dashboard، فكل شيء صحيح!**

---

## 📦 التحقق من أن كل شيء صحيح

### في Terminal:
```bash
# تحقق من أن الملفات مبنية
ls public/app.css    # يجب أن يكون 604 KB تقريباً
ls public/app.js     # يجب أن يكون 124 KB تقريباً

# تحقق من أن twilight.json صحيح
node -e "JSON.parse(require('fs').readFileSync('twilight.json', 'utf8')); console.log('✅ Valid');"
```

### في Dashboard:
عند إضافة قسم، يجب أن ترى:
- ✅ **"القسم الرئيسي (Hero) - سُفرة"** بأيقونة 🍽️
- ✅ **"قسم المنتجات (Grid/List/Menu) - سُفرة"** بأيقونة 🍕

إذا ظهرت هذه المكونات، **الثيم يعمل بشكل صحيح!** ✨

## ❓ الأسئلة الشائعة

**س: لماذا لا أرى مكونات سُفرة في الصفحة تلقائياً؟**
ج: لأن Salla Twilight يعمل بنظام المكونات - كل تاجر يختار ما يريد عرضه.

**س: هل يمكنني إضافة مكونات سُفرة بدون Dashboard؟**
ج: لا، هذا التصميم المعتمد من سلة - المكونات تُضاف فقط من Dashboard.

**س: لماذا تظهر أخطاء في Console؟**
ج: أخطاء عادية في Preview Mode - تختفي في Production.

**س: كيف أختبر الثيم بشكل كامل؟**
ج: أضف المكونات من Dashboard ثم تصفح المتجر كزائر.
